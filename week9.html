Introduction

This week we set up an experiment to determine if design improvements to a certain starting page, in this case the week 6 article about accessibility, will drive traffic to a secondary page which is a library of accessible color options. Using last week's data as well as some informal user testing, we implemented design changes and created a redirect experiment with Google Optimize. After a week of traffic, we will determine if the design changes had any effect compared to baseline.

Overview of Design

As can be viewed on this blog by navigated to week 6's article, the baseline week 6 article includes a large orange button directing users to a page which shows text and background color combinations which comply with accessibility guidelines. The goal is for users to click this button and view this content. Last week, no user clicked this button. I believe this is for several reasons:

  • The button is presented as "optional" content
  • The content is not interesting to someone not involved in web development

I believe the design allows users to see that the button is there, but they are simply not interested. The design changes implemented in the alternate page attempt to remedy this.

Summary of Changes

The design changes attempt to make a page which displays color combinations interesting to the user. I attempted to this by re-writing some article content to introduce contrast errors, describe what they are, and then direct users to a page to see examples of contrast errors. The baseline page simply says "click here to see accessible color combinations". The new page will say "see examples of what we're talking about". I beelive this revised phrasing, and placing the link in the first paragraph of the article will create a more compelling and useful link for the user since the page is presented as sometihng to aid in the understanding of the article's content rather than optional side content.

Creating a Google Optimize Experiment

To test these changes side by side, we've created an experience test with Google Optimize. The test we've created will direct users to either the existing week 6 article, or the new design for the week 6 article. This will allow us to compare data side by side for the two web page designs over the experiment period, with each page recieving an equal number of page views.

Since we already have google analytics script inserted into every page on this blog, the process for adding Google Optimize is fairly straight forward. Google Optimize asks that we add additoinal script to the sites we are experimenting, and link the test sites to a our Google Anaytics property. This creates a linkage between the experiment, and the goal set up in Google Analytics which is clicking the link to the colors page.

Setting up the experiment in Optimize is also relatively easy. We supply Google with the URL's of the baseline page and the new page we are testing, and Optimize takes care of the rest.

Summary

We are testing an alternative web page design for week 6's article attempting to drive traffic. We've taken the input from self examanination and informal user testing to create an alternative to design. Google Optimize will help us test this new design by presenting it to users who navigate to week 6's article. We have the test infrastructure and web design in place, now we await traffic to determine which site performs better (if any).
